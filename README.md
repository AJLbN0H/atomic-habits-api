# Atomic Habits API: Сервис формирования полезных привычек

Habit Tracker API — это бэкенд-решение для сервиса по созданию и контролю привычек. Проект реализует концепцию «атомных привычек», помогая пользователям внедрять полезные действия в жизнь через систему вознаграждений и своевременные уведомления.

## Технологический стек

* **Core:** Python 3.13, Django 5.2.
* **API:** Django REST Framework (DRF).
* **Task Queue:** Celery + Redis.
* **Scheduling:** Django-celery-beat (для периодических напоминаний).
* **Integrations:** Telegram Bot API (уведомления).
* **Security:** JWT (SimpleJWT).
* **Documentation:** Swagger/ReDoc.

## Ключевой функционал

### 1. Умные напоминания (Telegram Integration)
Реализована интеграция с Telegram-ботом. Пользователь указывает свой `chat_id`, и система автоматически присылает уведомление в заданное время выполнения привычки. За логику рассылки отвечают периодические задачи Celery Beat.

### 2. Бизнес-логика привычек
Система поддерживает два типа привычек:
* **Полезные:** Те, что пользователь хочет внедрить.
* **Приятные:** Те, что могут выступать в качестве вознаграждения за выполнение полезной привычки.

Реализована логика связей: полезная привычка может быть связана с приятной либо иметь текстовое вознаграждение (но не всё сразу!).

### 3. Ролевой доступ и публичность
* **IsOwner:** Пользователь имеет полный доступ только к своим привычкам.
* **Public Feed:** Реализован отдельный эндпоинт для просмотра публичных привычек других пользователей (без возможности редактирования), что позволяет искать вдохновение у сообщества.

### 4. Оптимизация и масштабируемость
* **Пагинация:** Список привычек разбит на страницы (по 5 объектов) для экономии трафика.
* **CORS:** Настроены заголовки для корректной работы с фронтенд-приложениями.

## Установка и запуск

1. Клонируйте проект:
   ```git clone https://github.com/AJLbN0H/atomic-habits-api.git```

2. Настройте .env:
   * `SECRET_KEY`, `DEBUG`, `DB_NAME`, `DB_USER`...
   * `TELEGRAM_BOT_TOKEN` — токен вашего бота.
   * `CELERY_BROKER_URL=redis://127.0.0.1:6379`

3. Разверните окружение:
   ```poetry install```
   ```python manage.py migrate```

4. Запустите воркеры и сервер в разных терминалах:
   ```celery -A config worker -l info```
   ```celery -A config beat -l info```
   ```python manage.py runserver```
